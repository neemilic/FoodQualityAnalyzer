# FoodQualityAnalyzer — Система анализа качества продуктов

## 1. Описание проекта
Приложение предназначено для мониторинга и анализа качества различных групп продуктов питания. Оно позволяет отслеживать сроки годности, рассчитывать интегральный показатель качества на основе уникальных параметров для каждого вида продуктов и визуализировать данные в виде графиков и отчетов.

## 2. Архитектура решения
Проект разделен на два основных модуля согласно принципу разделения ответственности:

*   **FoodQualityAnalyzer.Model (Class Library)** — библиотека классов, содержащая всю бизнес-логику, математические расчеты, интерфейсы и модели данных. Не зависит от графического интерфейса.
*   **FoodQualityAnalyzer.Desktop (WPF)** — графическое приложение, отвечающее за взаимодействие с пользователем, отображение диаграмм и управление файловой системой.

### Схема классов (ООП):
*   **`FoodProduct` (Abstract Class)**: Базовый класс для всех продуктов.
    *   *Свойства*: Название, срок годности, дата производства.
    *   *Методы*: Абстрактный метод `GetQuality()`, который переопределяется в наследниках.
*   **Наследники**: `Vegetable`, `Fruit`, `Meat`, `Bakery`. Каждый имеет по 2 уникальных параметра (например, для мяса — процент жирности и температура хранения), влияющих на расчет качества.
*   **`FoodQualityAnalyzer` (Main Logic Class)**:
    *   Реализован как **partial класс**.
    *   Одна часть отвечает за хранение и базовую логику.
    *   Вторая часть реализует интерфейс `IShrinkable` (удаление элементов).

## 3. Соответствие критериям (на 10 баллов)
В проекте реализованы следующие продвинутые механизмы C#:

*   **Интерфейсы (более 2-х)**:
    *   `ISpreadable`: Методы для добавления и группировки продуктов.
    *   `IShrinkable`: Методы для гибкого удаления продуктов из списков.
    *   `IStatistic`: Методы для расчета статистических показателей (медиана, среднее, min/max).
*   **Абстрактные классы (2 и более)**: `FoodProduct` (база для продуктов) и `BaseSerializer` (база для JSON/XML сериализации).
*   **Делегаты и События**: Использование `Action<T>` для уведомления интерфейса об обновлении данных в модели без прямой связи (Loose Coupling).
*   **Генерики (Generics)**: Обобщенный метод поиска и фильтрации `FilterBy<T>(Predicate<T> criteria)`.
*   **Перегрузка операторов**: Перегружен оператор `>` и `<` для класса `FoodProduct`, что позволяет сравнивать продукты по уровню их качества.
*   **Сериализация**: Реализована поддержка **JSON** (через Newtonsoft.Json) и **XML**. Архитектура позволяет переключать формат «на лету» с автоматической конвертацией файлов.

## 4. Инструкция пользователя (Интерфейс)
1.  **Главное окно**: Содержит список всех продуктов с чекбоксами.
2.  **Выбор продуктов**: При выборе одного или нескольких пунктов активируется кнопка «Показать качество».
3.  **Окно анализа**: Открывается новое окно с круговой диаграммой (PieChart), отображающей распределение качества выбранных продуктов.
4.  **Создание продукта**: Кнопка «Добавить» открывает форму, где можно выбрать тип продукта (Meat/Fruit и т.д.) — поля ввода меняются динамически в зависимости от выбора.
5.  **Настройки**: Выпадающий список в главном меню позволяет выбрать формат сохранения (JSON/XML). При смене формата программа переносит данные из старых файлов в новые.

## 5. Принципы SOLID
*   **Single Responsibility**: Логика расчетов отделена от логики сохранения файлов.
*   **Open/Closed**: Новые виды продуктов (например, `Drinks`) добавляются путем наследования от `FoodProduct` без изменения существующего кода анализатора.
*   **Liskov Substitution**: Любой объект-наследник (Мясо, Фрукты) может быть обработан анализатором как базовый `FoodProduct`.
*   **Interface Segregation**: Разделение на `ISpreadable` и `IShrinkable` позволяет классам не реализовывать лишние методы.
*   **Dependency Inversion**: Интерфейс зависит от абстракций, а не от конкретных классов продуктов.
